## What I did
Linux Journey — users, files, directories, and output redirection

## Commands I learned
echo "Hello, World!" — prints text to the terminal (like print in Python)
whoami — prints your current username
id — shows uid (User ID), gid (Group ID), and groups you belong to
id -un — prints only your username
cd .. — goes up one directory level
cd /home/project/things — absolute path navigation (starts from root /)
touch filename — creates an empty file
echo "text" > file.txt — writes text into a file, creates it if it doesn't exist
ls -l — long format listing, shows permissions, owner, size, date
ls -a — shows hidden files (files starting with a dot)

## What I learned
root is the superuser — has access to everything on the system.
Hidden files in Linux start with a dot (.) — ls won't show them unless you use -a.
Absolute paths start from / and give the full location of a file.
.. means the directory above your current one.
The > symbol redirects output into a file instead of printing to screen.

## What confused me
uid, gid, groups — the difference between them and why they exist.
